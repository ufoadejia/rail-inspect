# 铁轨探伤智能检修系统 — 开发方案

> 软件杯参赛项目 · 基于多模态大模型技术的设备检修知识检索与作业系统
> 落地方向：钢轨探伤

## 一、项目定位

面向铁路探伤工的「**拍图即识别 → 知识即检索 → 工单即作业**」一体化智能终端系统。
用多模态大模型解决探伤现场三大痛点：

1. **看图判伤难**：依赖经验，新手判伤一致性差 → 多模态大模型自动识别+判级
2. **查规程慢**：纸质手册翻找低效 → 跨模态 RAG 秒级检索，答案带溯源
3. **作业不规范**：工单流转靠纸质/口头 → 线上化工单闭环 + 自动生成报告

## 二、赛题要素对应

## 五、核心模块设计

### 5.1 多模态缺陷识别
- **方法**：Qwen-VL + few-shot 提示工程（无需训练）
- **输入**：钢轨表面照片 / 超声波 A·B·C 扫图
- **输出**：缺陷类型（8类）+ 当量 dB + 判级（正常/轻伤/重伤）+ 描述
- **判级**：按规程阈值规则判级（核伤 26/30dB、螺孔 24/28dB 等）
- **专业图像优化**：A 扫图先用传统 CV 做 ROI 提取再喂大模型（稳定性）
- **缺陷分类体系**：核伤、螺孔裂纹、纵向裂纹、剥离、磨耗、波磨、焊缝缺陷、表面擦伤

### 5.2 跨模态知识检索（RAG）
知识库构建：
```
原始资料(规程PDF/图谱/案例) → 类型路由
  ├─ 文本 → bge-m3 向量 → Milvus text_vec 列
  ├─ 图片 → CLIP 向量 → Milvus image_vec 列
  └─ 表格 → 自然语言描述 → 文本向量
```
检索流程：
```
query(文字/图片/组合)
  ├─ 文字 → bge 向量 → 文本召回
  └─ 图片 → CLIP 向量 → 图像召回（以图搜案例）
→ RRF 融合 → 重排 → 大模型生成(带[编号]引用溯源)
```

### 5.3 作业工单系统
状态机：
```
待派发 → 已派发 → 作业中 → 待验收 → 已归档
                  ↑__________| 驳回回退
```
- 关联检测结果（一次工单可含多次识别记录）
- 验收归档后一键导出 Word 探伤报告

### 5.4 探伤报告生成
自动汇总：基本信息表 + 检测结果表 + 处置建议（按判级给出换轨/监测/巡检建议）+ 落款，导出 .docx。

### 5.5 数据看板
工单状态分布、缺陷类型饼图、判级柱状图、关键统计指标。

## 六、数据策略

- **合成数据（已内置）**：`backend/data_pipeline/synthetic_data.py`
  - 6 条规程条款、4 条图谱描述、3 条判级表、3 条历史案例，共 16 条
  - 内容参照 TB/T 2975 公开资料整理，仅用于演示链路
- **真实数据接入**：在 `knowledge/raw/` 放 PDF/Word/图片，扩展 `build_knowledge.py` 的解析函数
  - PDF/Word → unstructured 解析 → 语义分块
  - 图片 → Qwen-VL 生成描述 + CLIP 向量
  - 表格 → 结构化 JSON + 自然语言描述

## 八、快速启动

```bash
# 1. 基础设施
cp .env.example .env
docker compose -f deploy/docker-compose.yml up -d

# 2. 后端
cd backend
pip install -r requirements.txt
python data_pipeline/build_knowledge.py   # 构建知识库
uvicorn app.main:app --reload --port 8000

# 3. 前端
cd frontend
npm install
npm run dev
```
- 后端 API 文档：http://localhost:8000/docs
- 前端：http://localhost:5173
- 无 DASHSCOPE_API_KEY 时自动走 mock，不阻塞开发

## 十、差异化亮点（答辩重点）

1. **跨模态检索**：支持「拍张照片搜出历史相似缺陷案例」，多数队伍只做文本 RAG
2. **答案可溯源**：每条建议标注来自哪份规程第几条，工业可信
3. **离线私有化方案**：本地 Qwen2-VL + Milvus，数据不出局
4. **专业图像预处理**：对超声 A 扫图做 ROI 提取再识别，体现领域理解
5. **完整作业闭环**：识别→检索→建议→工单→报告，而非只做问答 demo
6. **降级容错**：无 GPU/API key 时 mock 运行、Milvus 不可用时内存检索降级，保证答辩稳定演示
